# ANN 环境调优计划

## 0. 文档定位

本文件记录 `Phase 2` 之后对经验非线性环境的调优计划。当前它不阻塞 `Phase 8` 主线；按最新冻结决策，应在 `SVAR` 主结果完成后作为 `Phase 9` 的补充模块推进。

## 1. 当前状态

| 项目 | 现状 |
|---|---|
| 输入信息集 | 当前 ANN 已与 SVAR 使用同一组输入，不是只看当期变量 |
| output gap 方程 | ANN 初版优于 SVAR |
| inflation 方程 | ANN 初版尚未优于 SVAR |
| 当前阶段定位 | 不进入 `Phase 8` 主结果 |
| 结论 | ANN 环境已跑通，但还未达到可作为主结果环境的标准 |

## 1.1 当前冻结安排

| 项目 | 冻结结论 |
|---|---|
| 是否阻塞 `Phase 8` | 否 |
| 正确进入时点 | `SVAR` 主反事实与主表主图完成之后 |
| 所属阶段 | `Phase 9` |
| 论文定位 | 优先作为补充结果或附录候选 |

## 2. 调优优先级

按用户确认，调优顺序冻结为：

| 优先级 | 任务 | 说明 |
|---|---|---|
| 1 | 在相同输入集下继续调 MLP | 先不改信息集，先改训练与结构 |
| 2 | 在相同输入集下换更稳的训练策略 | 如优化器、早停、验证切分、正则化 |
| 3 | 增加 1 期额外滞后 | 在不过度扩参前提下增强时序信息 |
| 4 | 改成 RNN / LSTM / GRU | 作为后续结构性升级 |
| 5 | 同时大幅增滞后并上 RNN | 暂不建议 |

## 2.1 进入执行前门槛

| 检查项 | 要求 |
|---|---|
| `Phase 8` 主结果完成 | 必须 |
| `SVAR` 主表主图已产出 | 必须 |
| 已确认需要 ANN 补充结果 | 必须 |

## 3. 具体待办

| 任务 | 目标 |
|---|---|
| 调整 MLP 隐层结构与激活函数 | 改善 inflation 方程拟合 |
| 尝试不同时间切分与早停机制 | 降低小样本不稳定性 |
| 比较 `tanh` / `relu` / 小型双隐层 | 找到更稳结构 |
| 评估是否加入额外 1 期滞后 | 检查对 inflation 方程是否有帮助 |
| 若 MLP 仍不理想，再上 RNN | 保持主线推进的同时控制复杂度 |

## 4. 执行原则

| 原则 | 说明 |
|---|---|
| 不阻塞主线 | `Phase 3/4/5` 可以先按 LQ benchmark 推进 |
| 保持可比较性 | 在调优早期尽量维持与 SVAR 相同输入集 |
| 先小改后大改 | 先调训练和小网络，再考虑 RNN |
| 时刻对照 Hinterlang | 它是经验环境设计的核心参考文献 |
| 先过门槛再升格 | 未通过 inflation 拟合与动态稳定性门槛前，不升格为论文主结果 |

## 5. 作为 Phase 9 补充结果的验收门槛

| 验收项 | 要求 |
|---|---|
| inflation 方程拟合 | 不应继续明显弱于 `SVAR` |
| output gap 方程拟合 | 至少保持当前优势或不显著退化 |
| 动态反事实 | 不能出现明显失稳或不合理漂移 |
| 写作可解释性 | 至少能给出拟合比较与局限说明 |

---

一句话总结：

ANN 环境已进入“可运行但待优化”状态；当前正确位置是 `Phase 9` 的补充模块，先完成 `Phase 8` 的 `SVAR` 主结果，再按 `MLP 调参 -> 训练策略 -> 适度增滞后 -> RNN` 的顺序推进。
