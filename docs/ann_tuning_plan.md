# ANN 环境调优计划

## 0. 文档定位

本文件记录 `Phase 2` 之后对经验非线性环境的调优计划。它不阻塞 `Phase 3` 主线，但必须在进入更深入的经验环境实验前继续推进。

## 1. 当前状态

| 项目 | 现状 |
|---|---|
| 输入信息集 | 当前 ANN 已与 SVAR 使用同一组输入，不是只看当期变量 |
| output gap 方程 | ANN 初版优于 SVAR |
| inflation 方程 | ANN 初版尚未优于 SVAR |
| 结论 | ANN 环境已跑通，但还未达到 Hinterlang 式的理想效果 |

## 2. 调优优先级

按用户确认，调优顺序冻结为：

| 优先级 | 任务 | 说明 |
|---|---|---|
| 1 | 在相同输入集下继续调 MLP | 先不改信息集，先改训练与结构 |
| 2 | 在相同输入集下换更稳的训练策略 | 如优化器、早停、验证切分、正则化 |
| 3 | 增加 1 期额外滞后 | 在不过度扩参前提下增强时序信息 |
| 4 | 改成 RNN / LSTM / GRU | 作为后续结构性升级 |
| 5 | 同时大幅增滞后并上 RNN | 暂不建议 |

## 3. 具体待办

| 任务 | 目标 |
|---|---|
| 调整 MLP 隐层结构与激活函数 | 改善 inflation 方程拟合 |
| 尝试不同时间切分与早停机制 | 降低小样本不稳定性 |
| 比较 `tanh` / `relu` / 小型双隐层 | 找到更稳结构 |
| 评估是否加入额外 1 期滞后 | 检查对 inflation 方程是否有帮助 |
| 若 MLP 仍不理想，再上 RNN | 保持主线推进的同时控制复杂度 |

## 4. 执行原则

| 原则 | 说明 |
|---|---|
| 不阻塞主线 | `Phase 3/4/5` 可以先按 LQ benchmark 推进 |
| 保持可比较性 | 在调优早期尽量维持与 SVAR 相同输入集 |
| 先小改后大改 | 先调训练和小网络，再考虑 RNN |
| 时刻对照 Hinterlang | 它是经验环境设计的核心参考文献 |

---

一句话总结：

ANN 环境已进入“可运行但待优化”状态，后续调优按 `MLP 调参 -> 训练策略 -> 适度增滞后 -> RNN` 的顺序推进。
