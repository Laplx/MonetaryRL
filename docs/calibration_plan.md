# Phase 2 校准与估计计划

## 0. 文档定位

| 项目 | 说明 |
|---|---|
| 阶段 | `Phase 2` |
| 文档用途 | 明确哪些对象需要通过数据估计、哪些参数可通过文献校准、环境方程与经验规则分别如何估计。 |
| 上游约束 | 遵守 `docs/model_spec.md` 与 `docs/data_spec.md`。 |
| 关键参考 | `Hinterlang and Tänzer (2021)`。 |

## 1. 本阶段要回答的核心问题

### 1.1 估计什么

本阶段不是只估计一类对象，而是估计两类不同对象：

| 对象 | 是否必须 | 用途 |
|---|---|---|
| 环境/转移方程 | 必须 | 构造 RL 与反事实的经济环境 |
| 经验 Taylor rule | 必须 | 作为反事实比较对象之一 |

因此，用户此前的问题可以明确回答为：

$$
\text{我们首先要估计环境，其次还要单独估计经验 Taylor 规则}
$$

两者不是替代关系，而是并行关系。

## 2. 参数分类

### 2.1 三类参数

| 类别 | 例子 | 获取方式 |
|---|---|---|
| 环境参数 | $A,B,\Sigma$ 或非线性函数参数 | 用数据估计 |
| 规则参数 | $\phi_\pi,\phi_x,\phi_i$ | 用数据回归估计 |
| 规范目标参数 | $\lambda_\pi,\lambda_x,\lambda_i,\beta,\pi^\ast,i^\ast$ | 文献校准 + 规范设定 |

### 2.2 为什么要分开

| 参数 | 含义 |
|---|---|
| 环境参数 | 描述经济如何从当前状态与政策动作转移到下一期状态 |
| 规则参数 | 描述央行在经验上如何对状态作出反应 |
| 规范目标参数 | 描述我们用什么福利/损失标准评价政策 |

如果把这三类参数混在一起，后续理论 benchmark、RL 训练和反事实分析就会混乱。

## 3. 环境估计总框架

### 3.1 双环境主线

本文明确采用双环境设计：

| 环境 | 作用 | 文献依据 |
|---|---|---|
| 线性环境 | 作为经验线性世界与 benchmark 的桥梁 | Hinterlang 的递归 SVAR |
| 非线性环境 | 作为 RL 展现相对优势的主环境 | Hinterlang 的 ANN 环境 |

### 3.2 环境估计的总体思路

环境以通胀和产出缺口为核心状态，政策利率作为输入。估计目标是

$$
\text{给定 } (\pi_t, x_t, i_t \text{ 及其滞后}) \Rightarrow (\pi_{t+1}, x_{t+1})
$$

因此，环境估计关注的是状态转移，而不是政策反应函数本身。

## 4. 线性环境：递归 SVAR 计划

### 4.1 参考 Hinterlang 的递归结构

参考 Hinterlang，线性环境采用递归 SVAR，设定为：

$$
x_t = C_x + a_{xx,1}x_{t-1} + a_{x\pi,1}\pi_{t-1} + a_{xi,1}i_{t-1} + a_{xi,2}i_{t-2} + \varepsilon^x_t
$$

$$
\pi_t = C_\pi + a_{\pi x,0}x_t + a_{\pi x,1}x_{t-1} + a_{\pi x,2}x_{t-2}
+ a_{\pi\pi,1}\pi_{t-1} + a_{\pi\pi,2}\pi_{t-2} + a_{\pi i,1}i_{t-1} + \varepsilon^\pi_t
$$

其经济含义是：

| 设定 | 含义 |
|---|---|
| 输出缺口方程不含当期通胀 | 递归排序假设的一部分 |
| 通胀方程可含当期输出缺口 | 需求压力当期影响通胀 |
| 利率以滞后方式进入状态方程 | 与货币政策传导时滞一致 |

### 4.2 估计方法

| 步骤 | 方法 |
|---|---|
| Step 1 | 从 SVAR(2) 起步 |
| Step 2 | 按显著性或信息准则删去不显著二阶滞后 |
| Step 3 | 采用逐方程 OLS 估计递归结构 |
| Step 4 | 保留残差协方差，用于构造冲击方差 |

### 4.3 为什么采用递归 SVAR

| 理由 | 说明 |
|---|---|
| 与 Hinterlang 完全对齐 | 便于文献比较 |
| 容易解释 | 能直接读出状态转移系数 |
| 易于转成 RL 环境 | 估计后可直接写成模拟器 |

### 4.4 与 benchmark 的关系

需要明确：

| 对象 | 性质 |
|---|---|
| `Phase 1` 中的 LQ benchmark | 规范性理论 benchmark |
| 这里估计的递归 SVAR | 经验线性环境 |

二者不是同一个对象，但可以通过线性状态空间形式建立映射。后续代码里应当保持“理论 benchmark”和“经验线性环境”两个模块可分离。

## 5. 非线性环境：ANN 计划

### 5.1 基本思想

非线性环境估计的是

$$
x_t = f_x(\text{滞后状态与利率}) + \varepsilon^x_t
$$

$$
\pi_t = f_\pi(\text{当期/滞后状态与利率}) + \varepsilon^\pi_t
$$

这里的 $f_x,f_\pi$ 用神经网络近似。

### 5.2 结构选择

主线采用与 Hinterlang 接近的监督学习思路：

| 项目 | 计划 |
|---|---|
| 任务类型 | 回归 |
| 目标变量 | 当期或下一期通胀、产出缺口 |
| 输入变量 | 与 SVAR 使用相同的变量集，确保公平比较 |
| 网络类型 | 先用简单 MLP；若需要显式时序结构，再考虑 NARX 风格封装 |
| 比较标准 | MSE、样本内拟合、滚动样本预测稳定性 |

### 5.3 为什么不拘泥于 Matlab NARX

Hinterlang 用的是 Matlab 的单隐层 NARX。本文实现时不必机械复制工具，但必须保留三点：

| 必须保留的东西 | 说明 |
|---|---|
| 输入集合一致 | 确保 SVAR 与 ANN 比较公平 |
| 网络尽量简洁 | 避免样本量不大时过拟合 |
| 结果可解释 | 后续要能画 partial dependence 或局部响应图 |

### 5.4 初始网络建议

| 项目 | 建议 |
|---|---|
| 网络规模 | 单隐层或双隐层小网络 |
| 节点数搜索 | 先在 2-10 或 2-16 范围内做网格搜索 |
| 激活函数 | `tanh` 或 `ReLU` |
| 训练目标 | 最小化验证集 MSE |
| 早停 | 必须使用 |

## 6. 环境残差与冲击构造

### 6.1 为什么需要残差

RL 环境不能只依赖点预测，还需要随机性。环境冲击来自估计残差。

### 6.2 构造方式

| 环境 | 冲击构造 |
|---|---|
| SVAR | 使用方程残差方差或协方差 |
| ANN | 用样本残差分布估计噪声尺度 |

在实现时，基准做法为

$$
\varepsilon_t \sim \mathcal{N}(0,\hat{\Sigma}_\varepsilon)
$$

后续若发现残差明显非正态，可在稳健性阶段讨论 bootstrap 或经验抽样。

## 7. 经验 Taylor rule 估计计划

### 7.1 规则形式

经验 Taylor rule 的基准规格冻结为

$$
i_t=\alpha+\phi_\pi \pi_t+\phi_x x_t+\phi_i i_{t-1}+u_t
$$

### 7.2 估计目的

| 目的 | 说明 |
|---|---|
| 不是构造环境 | Taylor rule 描述的是历史政策反应，不是经济转移机制 |
| 是构造比较对象 | 后续反事实中，用它和 RL 规则、理论规则、历史政策做对照 |

### 7.3 扩展规格

可选增加：

| 规格 | 用途 |
|---|---|
| 无利率平滑项版本 | 与 TR93 更接近 |
| 用通胀偏离和利率偏离表示 | 与模型中心化形式更一致 |
| 分段或危机前后异质系数 | 仅作扩展，不是主线 |

## 8. 规范目标参数的校准计划

环境与规则估计以数据为主，但央行损失参数属于规范设定，需要单独校准。

### 8.1 需要校准的对象

| 参数 | 含义 |
|---|---|
| $\lambda_\pi$ | 通胀稳定权重 |
| $\lambda_x$ | 产出稳定权重 |
| $\lambda_i$ | 利率平滑权重 |
| $\beta$ | 折现因子 |
| $\pi^\ast$ | 通胀目标 |
| $i^\ast$ | 稳态或中性利率 |

### 8.2 获取方式

| 参数 | 获取方式 |
|---|---|
| $\pi^\ast$ | 先固定为 2% |
| $i^\ast$ | 由样本均值、稳态经验或文献给定 |
| $\beta$ | 文献常用值 |
| $\lambda$ 权重 | 文献校准 + 规范设定，后续做稳健性分析 |

### 8.3 为什么不能全靠数据估计

这些参数属于“目标函数”而非“环境规律”。它们反映的是研究者给定的规范性评价标准，不是单纯从历史数据中识别出来的客观参数。

## 10. 需要用户协助的触发条件

在以下情况下，应立刻请求用户协助：

| 触发条件 | 需要用户提供什么 |

| 项目 | 当前状态 |
|---|---|
| FRED 基础数据 | 大概率可识别，但本地自动下载可能受限 |
| Hinterlang 方法细节 | 已足够支撑本阶段 |
| 额外关键论文参数表 | 暂无硬性阻塞 |
