## 2. 研究定位冻结

| 主题 | 冻结结论 |
|---|---|
| 论文主线 | 本文将强化学习视为货币政策最优控制问题的数值求解器。 |
| 理论对照对象 | 经典最优控制方法，具体包括离散时间 LQ benchmark 下的 Bellman/Riccati 解。 |
| 研究问题 | 在可解析情形下比较 RL 与理论最优规则的一致性；在不可解析或难解析情形下检验 RL 的求解能力与政策结构。 |
| 非主线 | 不把 RL 作为居民、企业或多主体的学习行为模型；`Chen Mingli` 这一类文献只用于边界区分。 |
| 方法定位 | 先有理论 benchmark，再有 RL；不能跳过 benchmark 直接进入非线性扩展。 |
| 实证定位 | 通过现实参数与历史数据校准模型，并估计经验 Taylor rule，用于反事实模拟与福利比较。 |

## 3. 本文规范分析框架冻结

### 3.1 时间与频率

| 项目 | 冻结结论 | 原因 |
|---|---|---|
| 时间框架 | 离散时间 | 便于与季度宏观数据、Riccati 方程和常用 RL 算法统一。 |
| 基准频率 | 季度 | 与货币政策文献和经验数据最一致。 |
| 目标问题 | 无限期贴现损失最小化 | 与动态最优控制和 RL 折现回报最大化完全对应。 |
| RL 近似方式 | 用有限长度 episode 近似无限期问题 | 这是实现近似，不改变理论目标。 |

### 3.2 决策者与环境

| 项目 | 冻结结论 |
|---|---|
| 决策者 | 代表性中央银行。 |
| 控制工具 | 名义政策利率。 |
| 环境 | 一个由状态转移方程驱动的随机宏观环境，至少包含通胀与产出缺口。 |
| 环境性质 | benchmark 中为线性随机环境；扩展阶段允许非线性与约束。 |
| 解释层次 | 属于规范性政策分析，不讨论私人部门作为 RL 主体的学习路径。 |

### 3.3 变量表示方式

为统一理论推导、数值求解与 RL 实现，本文冻结如下表示方式。

| 变量 | 符号 | 定义 |
|---|---|---|
| 通胀偏离 | $\tilde{\pi}_t$ | 实际通胀相对目标通胀的偏离，即 $\tilde{\pi}_t=\pi_t-\pi^\ast$。 |
| 产出缺口 | $x_t$ | 实际产出相对潜在产出的缺口。 |
| 利率偏离 | $\tilde{i}_t$ | 名义政策利率相对稳态或中性利率的偏离，即 $\tilde{i}_t=i_t-i^\ast$。 |
| 上期利率偏离 | $\tilde{i}_{t-1}$ | 用于处理利率平滑项。 |
| 状态向量 | $s_t$ | 见下文冻结定义。 |
| 控制变量 | $a_t$ | 规范写法下记为当期利率偏离 $a_t=\tilde{i}_t$。 |

说明：

| 事项 | 规则 |
|---|---|
| 理论推导 | 默认使用中心化变量，即围绕目标通胀与稳态利率写模型。 |
| 经验估计 | 可先在原始单位下估计，再转换成中心化规格用于模型。 |
| RL 输入 | 可对状态进一步标准化，但经济学汇报必须回到原始或中心化经济单位。 |

## 4. 基准 benchmark 结构冻结

### 4.1 基准状态定义

本文的 benchmark 使用增广状态：

$$
s_t =
\begin{bmatrix}
\tilde{\pi}_t \\
x_t \\
\tilde{i}_{t-1}
\end{bmatrix}
$$

这是 `Phase 0` 最关键的冻结决策之一。

### 4.2 冻结该状态表示的原因

| 原因 | 说明 |
|---|---|
| 利率平滑项需要状态扩充 | 若单期损失包含 $(i_t-i_{t-1})^2$，则上期利率必须进入状态。 |
| 与 LQ 框架兼容 | 加入滞后利率后，状态转移仍可保持线性，损失仍可保持二次型。 |
| 与 RL 环境兼容 | RL 直接接受该状态向量作为 observation，可与理论 benchmark 一一对齐。 |
| 便于解释 | 最终政策规则可直接写成对通胀、产出缺口和利率惯性的反馈。 |

### 4.3 基准控制定义

benchmark 中的动作定义为

$$
a_t = \tilde{i}_t
$$

对应的实际名义利率为

$$
i_t = i^\ast + a_t
$$

说明：

| 项目 | 规则 |
|---|---|
| benchmark 中的理论最优控制 | 直接对 $a_t$ 求解。 |
| RL 中的动作空间 | 默认输出 $a_t$，如需数值稳定可在内部做缩放。 |
| ZLB 扩展 | 在扩展阶段通过对实际利率 $i_t$ 加约束实现，而不是改变 benchmark 动作定义。 |

## 5. benchmark 状态转移冻结

### 5.1 线性随机状态转移

本文 benchmark 的标准写法冻结为

$$
s_{t+1} = A s_t + B a_t + \Sigma \varepsilon_{t+1}
$$

其中

$$
\varepsilon_{t+1} \sim \mathcal{N}(0, I)
$$

且 $A,B,\Sigma$ 在 `Phase 2` 校准。

### 5.2 分量形式

等价地，可写成

$$
\tilde{\pi}_{t+1} = \alpha_{\pi\pi}\tilde{\pi}_t + \alpha_{\pi x}x_t + \alpha_{\pi i}\tilde{i}_t + \sigma_\pi \varepsilon^\pi_{t+1}
$$

$$
x_{t+1} = \alpha_{x\pi}\tilde{\pi}_t + \alpha_{xx}x_t + \alpha_{xi}\tilde{i}_t + \sigma_x \varepsilon^x_{t+1}
$$

$$
\tilde{i}_t = a_t
$$

因此第三个状态分量在下一期自然更新为当期动作。

### 5.3 转移结构的实现约束

| 约束 | 说明 |
|---|---|
| benchmark 必须线性 | `Phase 3-6` 的理论对照必须建立在线性 benchmark 上。 |
| 冲击先用高斯白噪声 | 相关冲击、异方差等只作为稳健性扩展。 |
| 状态维度先固定为 3 | benchmark 阶段不加入额外观测变量，避免对照失真。 |
| 动作只影响当期转移 | 不额外引入隐藏控制状态。 |

## 6. 央行目标函数冻结

### 6.1 benchmark 单期损失

benchmark 单期损失冻结为

$$
\ell_t = \lambda_\pi \tilde{\pi}_t^2 + \lambda_x x_t^2 + \lambda_i (\tilde{i}_t-\tilde{i}_{t-1})^2
$$

其中 $\lambda_\pi,\lambda_x,\lambda_i > 0$，具体数值在 `Phase 2` 校准。

### 6.2 长期目标

中央银行最小化折现总损失：

$$
V(s_t)=\min_{\{a_{t+k}\}_{k\ge 0}} \mathbb{E}_t \left[\sum_{k=0}^{\infty}\beta^k \ell_{t+k}\right]
$$

其中 $\beta \in (0,1)$ 为折现因子。

### 6.3 为什么采用该规格

| 项目 | 说明 |
|---|---|
| 通胀项 | 对应央行稳定价格目标。 |
| 产出缺口项 | 对应产出稳定目标。 |
| 利率平滑项 | 对应开题报告中“利率平滑偏好”，也是经验货币政策文献中的常见设定。 |
| 二次损失 | 与 LQ 最优控制一致，可提供理论 benchmark。 |

### 6.4 明确排除的做法

在 benchmark 中，以下设定不允许提前引入：

| 不允许事项 | 原因 |
|---|---|
| 非对称损失 | 会破坏 benchmark 的 LQ 结构，应留到扩展阶段。 |
| ZLB 约束 | benchmark 先做无约束情形，便于与 Riccati 解对照。 |
| 多目标非二次惩罚 | 会使 benchmark 不再是标准理论基准。 |

## 7. benchmark 理论输出冻结

在 benchmark 下，后续 `Phase 1/4` 必须产出以下理论对象：

| 对象 | 冻结要求 |
|---|---|
| Bellman 方程 | 必须写出。 |
| 二次型值函数猜测 | 必须写出。 |
| 离散 Riccati 方程 | 必须写出并求解。 |
| 线性反馈规则 | 必须写成 $a_t=-F s_t$ 的形式。 |
| 闭环稳定性检查 | 必须报告。 |

这意味着 benchmark 的实现必须能映射到标准离散时间 LQ 控制问题。

## 8. RL 问题映射冻结

### 8.1 MDP 对应关系

本文将规范性最优控制问题映射为 MDP：

| MDP 元素 | 冻结定义 |
|---|---|
| 状态 | $s_t=[\tilde{\pi}_t, x_t, \tilde{i}_{t-1}]^\top$ |
| 动作 | $a_t=\tilde{i}_t$ |
| 转移 | 由 benchmark 或扩展宏观环境给出 |
| 奖励 | $r_t=-\ell_t$ |
| 目标 | 最大化折现总回报，即最小化折现总损失 |

### 8.2 RL 在本文中的唯一合法角色

| 角色 | 允许与否 | 说明 |
|---|---|---|
| 数值最优控制求解器 | 允许 | 这是本文主线。 |
| 真实经济主体的学习模型 | 不作为主线 | 可在文献综述中讨论，但不进入主实验。 |
| 替代理性预期的家庭/企业学习机制 | 不进入本文主框架 | 这属于 `Chen Mingli` 一类方向。 |

### 8.3 benchmark 算法选择约束

| 项目 | 冻结结论 |
|---|---|
| 主算法 | PPO |
| 备选算法 | SAC |
| DDPG | 只作为文献对照参考，不作为首选实现路线 |
| 进入扩展阶段的前提 | PPO 在 benchmark 中稳定逼近理论最优 |

## 9. 反事实比较对象冻结

本文后续反事实与福利比较阶段，比较对象冻结为以下四类。

| 比较对象 | 是否必须 | 说明 |
|---|---|---|
| 理论最优规则 | 必须 | benchmark 中的规范基准。 |
| RL 学到的规则 | 必须 | 研究主对象。 |
| 经验 Taylor rule | 必须 | 现实数据中的经验政策规则。 |
| 历史实际政策路径 | 必须 | 用于现实比较。 |

可选比较对象：

| 对象 | 是否必须 | 说明 |
|---|---|---|
| 简单手工 Taylor 规则 | 可选 | 用于直观对照。 |
| 其他 RL 算法 | 可选 | 用于稳健性。 |

## 10. 评价指标冻结

后续所有主实验必须至少报告以下指标。

| 指标类别 | 必须内容 |
|---|---|
| 福利指标 | 折现总损失、平均单期损失 |
| 状态稳定性 | $\mathrm{Var}(\tilde{\pi}_t)$、$\mathrm{Var}(x_t)$ |
| 政策平滑性 | $\mathrm{Var}(\Delta i_t)$ 或相关平滑指标 |
| 轨迹对照 | 相同初值和冲击下的状态与政策路径 |
| 政策函数结构 | 线性系数、局部斜率、热图或切片图 |
| 训练稳定性 | 多随机种子结果分布、回报曲线 |

明确规定：

| 事项 | 规则 |
|---|---|
| 不能只报最优 loss | 必须解释政策函数结构与经济学含义。 |
| 理论与 RL 比较 | 必须共用同一损失函数与同一冲击设定。 |
| 稳健性 | 至少在种子维度做重复实验。 |

## 11. 扩展实验边界冻结

### 11.1 扩展顺序

扩展必须按下列顺序推进：

| 顺序 | 扩展内容 | 是否必须 |
|---|---|---|
| 1 | 非线性 Phillips 曲线或更一般的非线性状态转移 | 必须 |
| 2 | ZLB 约束 | 必须 |
| 3 | 非对称损失 | 可选 |

### 11.2 为什么这样排序

| 原因 | 说明 |
|---|---|
| 与开题报告一致 | 开题报告明确点名非线性 Phillips 曲线与 ZLB。 |
| 与论文结构一致 | 先在无约束非线性下检验，再进入政策约束。 |
| 与实现难度一致 | 非线性转移比约束优化更适合先建立扩展框架。 |

### 11.3 扩展前提

只有在以下条件满足后，才允许进入扩展：

| 条件 | 要求 |
|---|---|
| benchmark 已完成 | 理论最优控制已求出。 |
| RL baseline 已稳定 | PPO 可稳定训练。 |
| 理论与 RL 对照图表已产出 | 能明确说明二者接近程度。 |

## 12. 经验分析边界冻结

### 12.1 数据相关的结构性决策

| 项目 | 冻结结论 |
|---|---|
| 频率 | 季度数据 |
| 变量 | 通胀、产出缺口、政策利率为最小集合 |
| 经验规则形式 | 包含通胀、产出缺口和利率平滑项的 Taylor rule |
| 角色 | 用于校准、规则比较与历史反事实 |

### 12.2 本文对经验转移的解释边界

后续若使用历史数据估计转移方程并做反事实，必须在正文中明确声明：

$$
\text{政策规则改变} \Rightarrow \text{转移方程未必保持不变}
$$

即存在 Lucas critique 风险。本文将该类经验反事实解释为在给定转移近似下的政策比较，而非完整结构政策评估。

这条边界在 `Phase 0` 冻结，不允许后续写作中模糊处理。
